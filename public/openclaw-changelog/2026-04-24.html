GPT-5.5 landed, image generation got serious, local workflows got less annoying, and the release notes were basically a long apology to anyone who has ever debugged channels at midnight.

    Release headline

    GPT-5.5 became a first-class OpenClaw route

    • 2026.4.23 added GPT-5.5 catalog work through bundled Pi/OpenAI Codex metadata
    • Codex OAuth defaults stay on the `openai-codex/gpt-5.5` route instead of drifting to API-key OpenAI
    • Codex harness fixes covered approvals, app-server selection, model catalogs, Windows shims, replay, verbose progress, OAuth status
    • Translation: fewer “why is Codex being weird?” support spirals
    This is the least glamorous slide and probably the most useful one.

    Image + voice surface

    Media tools stopped being side quests

    • OpenAI `gpt-image-2` became the bundled image-gen default with 2K/4K size hints
    • OpenAI image generation/editing now works through Codex OAuth — no `OPENAI_API_KEY` required
    • OpenRouter image models joined `image_generate`

    OpenClaw is quietly turning “send media to an agent” from demo magic into plumbing.

    Operator workflow

    Local mode and model adding got real

    • `/models add <provider> <modelId>` lets users register a model from chat without a Gateway restart
    • Local embedded TUI mode runs terminal chats without Gateway while keeping plugin approval gates
    • Auto-install missing provider/channel plugins during setup
    • Diagnostics export bundles sanitized logs, status, health, config, and stability snapshots
    Less yak shaving. More “try the model and move on with your life.” Radical technology.

    Channels

    The messy chat surfaces got attention

    • Telegram markdown images now become media replies instead of ugly plain-text URLs
    • Slack MPIMs are treated as group chat context, and verbose tool traces stop leaking into rooms
    • WhatsApp setup avoids eager Baileys loading; media normalization improved
    • BlueBubbles group prompts and iMessage sends/tapbacks got fixes
    • Discord, Telegram, Slack preview streaming now shows tool progress before finalization
    This is where users feel quality. Not in the model list. In whether the bot embarrasses itself in a group chat.

    Security + trust

    The team kept tightening the doors

    • Owner-only commands now require owner identity instead of permissive fallback rules
    • External content strips common chat-template special tokens to prevent role-boundary spoofing
    • Mobile cleartext rules, Teams/Discord/QQBot auth paths, webhook SecretRefs, SSRF guards, and avatar routes all got hardening
    • Config edits for agents are fail-closed by default
    The security work is not one feature. It is a rhythm now.

    How Henry should say it

    • Lead with: “This week was about making OpenClaw less haunted.”
    • Then give three receipts: GPT-5.5 route, `/models add`, and channel/media fixes
    • Use the 2,481 commits / 167 authors stat early; it gives the whole segment weight
    • Do not over-explain every fix. Pick the ones people can feel: images, local TUI, Slack/Telegram/WhatsApp, security
    If someone asks “what actually shipped?”, avoid philosophy. Say names. Say commands. Say versions.

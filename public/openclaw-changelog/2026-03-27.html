The releases at a glance

What shipped in the official changelog

2026.3.22 — The Big One

  • ClawHub native skill install/update
  • Plugin SDK rewrite (openclaw/plugin-sdk/*)
  • Chrome extension relay removed
  • Legacy .moltbot and CLAWDBOT_* env removed
  • Matrix plugin rewritten with official SDK
  • OpenAI default model switched to gpt-5.4

2026.3.23 — Stabilization

  • Qwen/Alibaba Cloud Model Studio support
  • UI/UX polish across Control UI
  • CSP hardening for inline scripts
  • Auth and credential handling fixes
  • Bundled plugin runtime fixes

2026.3.24 — Polish + OpenAI API

  • Gateway now exposes /v1/models, /v1/embeddings, /v1/chat/completions
  • MS Teams SDK migration with AI-agent UX patterns
  • Skills got one-click install recipes
  • Slack interactive replies restored
  • Discord auto-thread naming via LLM
Official release window2
Biggest ecosystem win

ClawHub is now the default skills source

  • openclaw skills search|install|update now hits ClawHub first
  • Skills get tracked update metadata and gateway-side install support
  • This makes OpenClaw less "configure everything yourself" and more "install what you need"
ClawHub becoming the primary install source is a platform move. It means skills can be discovered, shared, and updated without manual config surgery.
Release 2026.3.223
Developer-facing

The plugin SDK got a real public surface

What changed

  • New public SDK: openclaw/plugin-sdk/*
  • Old openclaw/extension-api removed
  • Message discovery now via describeMessageTool(...)
  • Bundled plugins use injected runtime for host ops

Why it matters

  • Cleaner plugin authoring
  • Less internal leakage
  • Stable surface for external contributors
  • Migration guide: docs.openclaw.ai/plugins/sdk-migration
This is the "stop building on internal surfaces" cleanup that real platforms do before they scale plugin ecosystems.
Release 2026.3.224
Integration story

OpenClaw gateway now speaks OpenAI API

  • Gateway exposes /v1/models, /v1/embeddings, /v1/chat/completions, /v1/responses
  • Explicit model overrides forwarded correctly
  • Better compatibility with RAG tools and other OpenAI-expecting clients
If you want to point third-party tools at OpenClaw as if it were an OpenAI endpoint, this is the release that makes that real.
Release 2026.3.245
Channel spotlight

MS Teams got a proper SDK migration

  • Migrated to the official Microsoft Teams SDK
  • Streaming 1:1 replies, welcome cards, prompt starters
  • Feedback/reflection, typing indicators, native AI labeling
  • Message edit and delete support added
Teams is a big enterprise surface. This makes OpenClaw feel native there instead of bolted-on.
Release 2026.3.246
Cleanup theme

Legacy baggage got removed

Gone

  • Chrome extension relay and driver: "extension"
  • CLAWDBOT_* and MOLTBOT_* env names
  • ~/.moltbot state dir migration
  • nano-banana-pro skill wrapper
  • Old openclaw/extension-api

Why this matters

  • Smaller attack surface
  • Less config confusion
  • Simpler onboarding for new users
  • Signals the project is serious about not accruing debt
Removing legacy cruft is unglamorous work, but it is how you keep a fast-moving project from collapsing under its own history.
Release 2026.3.227
Control UI

Skills management got much better

  • Status-filter tabs: All / Ready / Needs Setup / Disabled
  • Click-to-detail dialog with requirements, toggle, install action, API key entry
  • One-click install recipes for bundled skills
  • /tools now shows what you can actually use right now
This is the kind of UX that makes people say "oh, this is actually usable" instead of "let me read the docs first."
Releases 2026.3.23 + 2026.3.248
Security and stability

The usual hardening continued

  • CSP hardening for inline scripts in Control UI
  • Exec sandbox blocks more dangerous env vars (MAVEN_OPTS, GLIBC_TUNABLES, etc.)
  • Gateway channel startup now isolates failures per-channel
  • Telegram, WhatsApp, Discord all got reliability fixes
  • Auth token handling fixed to stop reverting to stale credentials

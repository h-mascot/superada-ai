Architecture shift

Ollama, vLLM, SGLang are now provider plugins

  • Provider-owned onboarding and discovery
  • Model-picker setup and post-selection hooks
  • Core provider wiring is more modular
  • Sets the pattern for future provider additions
This is the same plugin architecture move we saw with channels. Local inference is no longer special-cased; it's a first-class plugin citizen.

Security sweep

12+ GHSAs addressed in one release

Device pairing hardening

  • Bootstrap tokens now single-use
  • Scopes capped to approved baseline
  • WebSocket preauth tightened

Exec approval hardening

  • Unicode obfuscation blocked
  • Shell payload ambiguity closed
  • Ruby/Perl/PowerShell wrappers handled

Other fixes

  • Workspace plugin auto-load disabled (requires explicit trust)
  • Gateway auth scope self-declaration blocked
  • Feishu/LINE/Zalo webhook hardening
  • Session visibility guards for sandboxed subagents

Release v2026.3.138 / 11
OpenClaw
Session management

Compaction logic refined

  • Token sanity checks now compare against full pre-compaction totals
  • Oversized toolResult payloads trimmed consistently
  • Heartbeat-only sessions no longer trigger compaction
  • Cache-ttl marker writes skipped after same-attempt compaction
  • Persona drift reduced via configurable custom instructions
Compaction bugs are subtle and annoying. These fixes prevent sessions from getting stuck, double-compacting, or losing token counts.
Releases v2026.3.12, v2026.3.139 / 11
OpenClaw
Breaking change

MEMORY.md now wins over memory.md

  • Agents now load at most one root memory bootstrap file
  • MEMORY.md takes priority; memory.md only used when absent
  • Fixes duplicate injection on case-insensitive Docker mounts

Action required

If you intentionally kept both files and depended on both being injected, merge them before upgrade.

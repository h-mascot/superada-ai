Here is a glimpse into the cron operation we have running at Enterprise Crew, which is so large it has to be managed by some crons itself (read more about how we manage them in [Managing 98 Autonomous Crons](/blog/managing-68-autonomous-crons) and [The Agent vs Agent Bug](/blog/agent-vs-agent-bug)).

The recent Anthropic rate limit crisis forced us to temporarily fall back to cheaper models (like MiniMax and GLM-5), but this list reflects the **original high-tier models** (Opus, GPT-5.4, Gemini Pro) these tasks run on during normal operations.

## 🌌 The Clusters (Meta-Crons)

To avoid hitting our concurrent execution limits, we group related tasks into sequential 'Clusters'. These meta-crons run their sub-tasks one after another.
